# ParkWatch AI вЂ” Parking Enforcement Intelligence Dashboard

AI-driven parking enforcement intelligence system built with React + Node.js. Visualises violation hotspots, hourly trends, and enforcement zone rankings derived from DBSCAN clustering and impact scoring of police violation records.

## Project Description
ParkWatch AI is a web application built to solve traffic congestion caused by illegal parking. Instead of having traffic officers drive around randomly hoping to find double-parked cars, the system processes about 300,000 police violation records from January to May to map out the actual trouble spots.

To do this, the Python data pipeline uses DBSCAN clustering. This algorithm groups the geographical coordinates of violations to identify dense hotspots while filtering out random, one-off events as noise. Once the clusters are identified, the system calculates a Congestion Impact Score for each hotspot. This score is computed by multiplying the total violation count of a cluster by the average resolution time (how long the vehicle blocked the road) and weighting it by the capacity of the roadway.

The backend is built with Node.js and Express. It serves this processed hotspot, timeline, and statistical data through a set of API endpoints. To provide a clear proof of concept that the application would work under live operational conditions, we also run the data in a simulated live format which enables live congestion highlights and thus operational mitigation protocols. For demonstration purposes, the system processes streaming telemetry, enabling real-time updates to metrics, active clock telemetry, and dispatch action indicators to prove the platform can handle live feeds in real-world scenarios.

The frontend is built using React and Vite. It features a Leaflet map where users can toggle between a raw violation density heatmap and a congestion impact heatmap, highlighting the difference between where violations simply occur versus where they actually choke traffic.

Hotspots are dynamically classified into categories like Metro Spillovers, Commercial Hubs, and Intersection Blockages, complete with quick-filter tabs so operators can prioritize zones easily. Additionally, the dispatcher card features operational action plan drawers. Clicking on a hotspot reveals a specific tactical mitigation plan, giving patrollers concrete instructions such as where to station tow trucks during peak hours, where to install physical medians etc.

## Quick Start

### 1. Start the backend

```bash
cd server
npm install
node index.js
```

The API server starts on **http://localhost:3001**.

### 2. Start the frontend

```bash
cd client
npm install
npm start
```

The Vite dev server starts on **http://localhost:5173** and proxies `/api/*` requests to the backend.

## API Endpoints

| Endpoint              | Description                                |
|-----------------------|--------------------------------------------|
| `GET /api/hotspots`   | All violation hotspot clusters             |
| `GET /api/stats`      | Headline stats & distribution breakdowns   |
| `GET /api/timeline`   | Hourly violation distribution (0вЂ“23)       |
| `GET /api/top-junctions` | Top 10 hotspots ranked by impact score  |

## Data

The `/output` directory contains pre-processed JSON files generated by the Python EDA pipeline from ~300,000 police violation records (JanвЂ“May). The backend reads these at startup.

## Stack

- **Frontend:** React 18, Vite, Leaflet.js, Recharts, Pure CSS
- **Backend:** Node.js, Express
- **Data:** DBSCAN clustering + impact scoring (Python pipeline, separate repo)
